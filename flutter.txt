1. key arguments by default are not required. To enforce key argument in function invocation a 'required' key word in front of parameter needed e.g. void method({required this.colors})
2. when a constant variable wil be initialised dynamically but the value is not yet known than that variable can be marked as final instead of const.
3. Theese 2 constructors can be used replaceble:
GradientContainer({super.key}); <=> GradientContainer({key}) : super(key: key) {}
4.  Command to generate KEY needed to release app in the Play Store
  keytool -genkey -v -keystore upload-keystore.jks -storetype JKS -keyalg RSA -keysize 2048 -validity 10000 -alias upload
5. Important file in the project: 
  - android/app/build.gradle.kts 
  - android/app/src/main/kotlin/com/example/rcatyb/MainActivity.kt 
  - android/key.properties - contains information about Key used for publishing the app in the Play Store
  - android/app/upload-keystore.jks  - the key used to publish the app in the Play Store
  - pubspec.yaml - list of packages used
6. To start emulator:
  /home/user/Android/Sdk/emulator/emulator -list-avds
  /home/user/Android/Sdk/emulator/emulator -avd  Medium_Phone_API_36.1
